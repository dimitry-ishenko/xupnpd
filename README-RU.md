## Обзор

**xupnpd** — это простой и портативный UPnP медиасервер, предназначенный для трансляции IPTV в домашней сети. Он позволяет DLNA-совместимым устройствам воспроизводить IPTV-контент, даже если они не имеют встроенной поддержки multicast ( multicast) или плейлистов из Интернета.

### Основные сценарии использования

*   **IPTV на роутере:** Установите xupnpd на свой домашний WiFi-роутер, чтобы смотреть интернет-телевидение на различных устройствах (Smart TV, игровых приставках, планшетах) без необходимости использования отдельного компьютера или STB-приставки от провайдера (при условии, что потоки не зашифрованы).
*   **Распределение спутникового ТВ:** Транслируйте сигнал с одного спутникового ресивера на несколько телевизоров в вашей домашней сети.
*   **Медиасервер:** Расшаривайте локальные видеофайлы с вашего роутера/ПК для DLNA-плееров.
*   **Агрегация онлайн-видео:** Воспроизводите контент с онлайн-платформ, таких как YouTube, Vimeo, Vkontakte и других, прямо на вашем телевизоре.

Не являясь DLNA-сертифицированным, xupnpd эффективно анонсирует списки unicast или multicast телеканалов с помощью механизмов UPnP/DLNA и совместим с широким спектром устройств.

## Возможности

*   Встроенный DLNA/UPnP медиасервер для анонсирования списков каналов или медиафайлов.
*   Поддержка HTTP-источников (списки URL-адресов интернет-стримов).
*   Встроенный multicast-прокси для IPTV от провайдера или поддержка внешнего `udpxy`.
*   Традиционный медиасервер для расшаривания видеофайлов в сети.
*   Поддержка YouTube, Vimeo, Vkontakte, IVI и других онлайн-платформ.
*   Воспроизведение видео в высоком разрешении (HD).
*   Интернет-радио.
*   Механизм расширения на основе Lua-скриптов для добавления новых источников видео.
*   **Без транскодирования** (только преобразование метода доставки).
*   Поддержка плейлистов M3U.
*   Планируемое автоматическое обновление фидов/списков каналов.
*   Группировка контента и отображение в виде дерева.
*   Списки контроля доступа (ACL) и родительский контроль.
*   Веб-интерфейс и внешний API для управления.
*   Открытое программное обеспечение.

## Требования

*   **Аппаратное обеспечение:**
    *   Роутер с чипсетом Atheros или Broadcom (ar71xx, mipsel и т.д.), например:
        *   D-Link DIR-320
        *   ASUS WL-500gP, N-16, RT-N56U
        *   Zyxel Keenetic
        *   TP-LINK WR841ND
    *   *Альтернатива:* ПК или любое другое устройство под управлением ОС на базе Linux или BSD.
*   **Программное обеспечение:**
    *   Кастомная прошивка, такая как **OpenWrt** или **DD-Wrt** (для роутеров).
    *   (Опционально) `udpxy` для внешнего преобразования multicast в HTTP.

## Установка и Быстрый старт

1.  **Скачайте** архив xupnpd на ваш роутер (используя `scp`, `sftp` или `ftp`).
2.  **Распакуйте** архив (например, `tar zxf xupnpd.tar.gz`).
3.  **Запустите** подходящий бинарный файл для вашей архитектуры:
    *   `xupnpd/xupnpd-mipsel`
    *   `xupnpd/xupnpd-ar71xx`
4.  **Откройте** веб-интерфейс в вашем браузере: `http://192.168.1.1:4044` (замените `192.168.1.1` на IP-адрес вашего роутера).
5.  **Загрузите** ваш IPTV-плейлист в формате M3U через веб-интерфейс.
6.  **Наслаждайтесь** IPTV на вашем DLNA-совместимом плеере. Ваш сервер должен отображаться как "UPnP-IPTV".

## Формат плейлиста (M3U8)

xupnpd использует плейлисты в формате M3U8 для описания структуры медиаконтента.

### Директивы EXTM3U (Заголовки плейлиста)

Строки, начинающиеся с `#EXTM3U`, могут содержать следующие атрибуты:
*   `name` — Название плейлиста.
*   `type` — Тип видео по умолчанию.
*   `dlna_extras` — DLNA-профиль по умолчанию.
*   `plugin` — Обработчик (расширение) по умолчанию для всего плейлиста (например, для YouTube, Vimeo).

### Директивы EXTINF (Записи каналов)

Строки, начинающиеся с `#EXTINF`, описывают отдельные каналы и могут содержать:
*   `logo` — URL логотипа канала (изображение JPEG).
*   `group-title` — Название группы (элементы с одинаковым значением группируются в одной папке).
*   `type` — Тип видео.
*   `dlna_extras` — DLNA-профиль.
*   `plugin` — Имя обработчика (расширения) для данного конкретного потока.

### Допустимые значения

*   **`type`:** `avi`, `asf`, `wmv`, `mp4`, `mpeg`, `mpeg1`, `mpeg2`, `ts`, `mp2t`, `mp2p`, `mov`, `mkv`, `3gp`, `flv`, `aac`, `ac3`, `mp3`, `ogg`, `wma`
*   **`dlna_extras`:** `none`, `mpeg_ps_pal`, `mpeg_ps_pal_ac3`, `mpeg_ps_ntsc`, `mpeg_ps_ntsc_ac3`, `mpeg1`, `mpeg_ts_sd`, `mpeg_ts_hd`, `avchd`, `wmv_med_base`, `wmv_med_full`, `wmv_med_pro`, `wmv_high_full`, `wmv_high_pro`, `asf_mpeg4_sp`, `asf_mpeg4_asp_l4`, `asf_mpeg4_asp_l5`, `asf_vc1_l1`, `mp4_avc_sd_mp3`, `mp4_avc_sd_ac3`, `mp4_avc_hd_ac3`, `mp4_avc_sd_aac`, `mpeg_ts_hd_mp3`, `mpeg_ts_hd_ac3`, `mpeg_ts_mpeg4_asp_mp3`, `mpeg_ts_mpeg4_asp_ac3`, `avi`, `divx5`, `mp3`, `ac3`, `wma_base`, `wma_full`, `wma_pro`

## Конфигурация (xupnpd.lua)

Основной файл конфигурации — `xupnpd.lua`. Вот пример с распространёнными параметрами:

```lua
-- Сетевой интерфейс для SSDP-анонсов (напр., 'eth0', 'br0', 'br-lan')
cfg.ssdp_interface='br0'

-- Разрешить хосту получать собственные multicast-анонсы (0 или 1)
cfg.ssdp_loop=0

-- HTTP-порт для входящих подключений (веб-интерфейс)
cfg.http_port=4044

-- Источник для логов ('syslog', 'local0' - 'local7')
cfg.log_facility='local0'

-- Запуск в режиме демона
cfg.daemon=true

-- Встроенный режим (без логов, без pid-файла)
cfg.embedded=true

-- Уровень трассировки SSDP (0-выкл, 1-базовый, 2-полный; требует cfg.daemon=false)
cfg.debug=1

-- URL для внешнего udpxy прокси (закомментируйте для использования встроенного прокси!)
--cfg.udpxy_url='http://192.168.1.1:4022'

-- Сетевой интерфейс (обычно WAN) для приёма multicast-потоков для встроенного прокси
cfg.mcast_interface='eth1'

-- Режим HTTP unicast прокси: 0-выкл, 1-прокси только радио, 2-прокси всё
cfg.proxy=2

-- Сетевой таймаут
cfg.http_timeout=15

-- Включить передачу DLNA extras
cfg.dlna_extras=true

-- Режим совместимости с XBox 360
cfg.xbox360=false

-- Включить UPnP/DLNA уведомления об изменениях
cfg.dlna_notify=true

-- Группировать контент по 'group-title'
cfg.group=true

-- Отображаемое имя устройства
cfg.name='UPnP-IPTV'

-- UUID устройства (если nil, генерируется новый при каждом запуске)
cfg.uuid='60bd2fb3-dabe-cb14-c766-0e319b54c29a'

-- Тип IPTV-потока по умолчанию
cfg.default_mime_type='mpeg'

-- Интервал обновления фидов в секундах (0 - отключить)
cfg.feeds_update_interval=0

-- Плейлисты и папки с локальными медиафайлами
-- Формат: { 'путь', 'Отображаемое имя', 'опционально;acl;список' }
playlist=
{
    { './playlists/mozhay.m3u', 'Mozhay.tv' },
    { './localmedia', 'Локальные медиафайлы', '127.0.0.1;192.168.1.1' } -- Ограничено ACL
}

-- Фиды из расширений
-- Формат: { 'имя_плагина', 'идентификатор_фида', 'Отображаемое имя' }
feeds=
{
    { 'vimeo',   'channel/hd',          'Vimeo HD Channel' },
    { 'vimeo',   'channel/hdxs',        'HD Xtreme sports' },
    { 'youtube', 'channel/top_rated',   'YouTube Top Rated' },
}
```

## Пример плейлиста (M3U8)

```m3u8
#EXTM3U name="Моё IPTV"
#EXTINF:0 logo=http://example.com/chan1.jpg type=mpeg dlna_extras=mpeg_ps_pal,Канал 1
udp://@234.5.2.8:20000
#EXTINF:0,Канал 2
http://192.168.1.1:4022/udp/234.5.2.2:20000
#EXTINF:0 group-title="Основные",Канал 3
http://020.example.tv
#EXTINF:0 logo=http://b.vimeocdn.com/ts/204/056/204056508_200.jpg type=mp4 plugin=vimeo,The Curious Fate of Humankind
http://vimeo.com/30381893
#EXTINF:0 logo=http://i.ytimg.com/vi/kffacxfA7G4/1.jpg type=mp4 plugin=youtube,Justin Bieber - Baby ft. Ludacris
http://www.youtube.com/watch?v=kffacxfA7G4
```
